# UiPath Integration Contract

This document defines the intended boundary between UiPath Automation Cloud / Maestro Case and the SwarmOps local agent layer.

## Track Target

UiPath AgentHack Track 1: UiPath Maestro Case.

## Integration Principle

UiPath should be the case orchestration and governance layer. SwarmOps should provide bounded agent analysis, risk flags, and evidence JSON. The submission should not claim deep UiPath integration until the flow is actually connected in Automation Cloud.

## Request From UiPath To SwarmOps

```json
{
  "case_id": "SO-CASE-001",
  "stage": "vendor_data_review",
  "request": "Onboard Northstar Components as a new vendor for a $48,500 rush purchase.",
  "vendor": {
    "name": "Northstar Components",
    "requested_purchase_amount": 48500,
    "urgency": "rush"
  },
  "requested_actions": [
    "normalize_request",
    "check_missing_fields",
    "draft_approval_summary",
    "build_evidence_report"
  ]
}
```

## Response From SwarmOps To UiPath

```json
{

  "risk_flags": [

```

## UiPath Case Stages

1. Intake
2. Vendor Data Review
3. Compliance Check
4. Human Approval
5. Fulfillment Handoff
6. Final Audit

## Human Approval Gate

Approval gate:

```text
Approve vendor setup after missing artifacts are collected
```

Blocked actions:

- approve vendor record
- accept compliance risk
- create purchase order
- send external vendor email
- finalize case

Allowed automatic actions:

- parse request
- classify urgency
- identify missing fields
- draft approval memo
- compile audit trail

## Demo Evidence Required

- Screenshot or video of UiPath case stages.
- Screenshot or video of human approval step.

- Clear statement of what is prototype-local and what is UiPath-connected.
